OWASP Top 10 - 2021

Learn about the most critical security risks to web applications

  • 1. Broken Access Control

    Exploits improper access restrictions, allowing unauthorized access to sensitive resources.

  • 2. Cryptographic Failures

    Focuses on weaknesses in protecting sensitive data, like insecure storage or transmission.

  • 3. Injection

    Occurs when untrusted input is processed as a command or query, such as SQL Injection.

  • 4. Insecure Design

    Highlights flaws in the design phase that lead to exploitable vulnerabilities.

  • 5. Security Misconfiguration

    Results from insecure default configurations, unnecessary files, or unprotected endpoints.

  • 6. Vulnerable and Outdated Components

    Using outdated libraries or components with known vulnerabilities.

  • 7. Identification and Authentication Failures

    Weak authentication mechanisms that attackers can exploit to impersonate users.

  • 8. Cross-Site Scripting (XSS)

    Enables attackers to inject malicious scripts into web pages viewed by other users.

  • 9. Insufficient Logging and Monitoring

    Lack of proper monitoring and logging allows attacks to go unnoticed.

  • 10. Server-Side Request Forgery (SSRF)

    Enables attackers to send unauthorized requests on behalf of the server.
